Name

pls - lists the files in a directory according as Pipeline file sequences

Synopsis

pls [options] [directory]

pls --help
pls --html-help
pls --version
pls --release-date
pls --copyright
pls --license

Description

A utility program which lists the files in a directory grouped by Pipeline file sequence.  If a directory name is given (directory), then the contents of that directory will be listed.  If no directory option is given, the current directory will be listed.  This tool is a more practical replacement for ls(1) when listing directories containing large numbers of images or other sequentially named files.

A file is considered to be part of a Pipeline file sequence if it contains one or more digits ("0", "1" ... "9") and shares a common prefix and/or suffix with one or more files in the directory.  In other words, files of the form:

[prefix]frame-number[suffix]
Where (suffix) does not contain any digits, the (frame-number) contains only digits and the (prefix) contains any character as long as the last character is not a digit.  The (frame-number) can be zero-padded or unpadded.

The default output of pls(1) lists file sequences in alphabetic order in the following format:

[prefix]pad[suffix]  (start-endxby[: range[,range[...]]])

Where (pad) is one or more "@" characters, each representing a padded frame number digit.  The (pad) component can also be a "#" character which is shorthand for 4-digit padded frame numbers.  In other words, "#" means the same thing as "@@@@".

The frame range pattern is composed of the start frame (start), the end frame (end) and the frame increment (by).  If there are missing frames in the sequence, the frame range patten will be followed by a list of existing frame ranges (range) which will either be single frame numbers or contigous frame ranges.

As an example, compare the output from ls(1) and pls(1) for a sample directory:

> ls
comp.006.rgb
comp.112.rgb
comp.114.rgb
comp.115.rgb
comp.116.rgb
render.0001.tif
render.0002.tif
render.0004.tif
render.0005.tif
render.0006.tif

> pls
comp.@@@.rgb (6-116x1: 6,112,114-116)
render.#.tif (1-6x1: 1-2,4-6)

> pls --fcheck
fcheck -n 6 116 1 comp.@.rgb
fcheck -n 1 6 1 render.#.tif

Information Options

The following options provide information about pls(1) and then immediately exit:

--help
Prints a brief usage message and exits.
--html-help
Opens this page using the default web browser.
--version
Prints the version number of this software and exits.
--release-date
Prints date that this software was released and exits.
--copyright
Prints the copyright notice for this software and exits.
--license

Prints the license agreement for this software and exits.

Global Options

--fcheck
--fcheck-exec
The --fcheck option prints an alternative output format consisting of a set of fcheck(1) command lines which which display the file sequences (if they are legal image files).  The --fcheck-exec option will execute these fcheck command lines in addition to printing them.
--zero
--size=bytes

In addition to the normal output, these two options cause pls(1) to print the frame numbers of files which are zero length (--zero) or are smaller than the given size (--size=bytes).  This can be useful for detecting bad image frames in a sequence.  The bad frame numbers are listing at the end of the line between "<BAD:" and ">" seperators.

These two options are mutually exclusive and cannot be used with the --fcheck or --fcheck-exec option.
--stats=level 
Print stage statistics with the given level of vebosity (level). 

By default, stage statistics are disabled.

--warnings=level

Print warning messages with the given level of vebosity (level).

By default, warning messages are disabled.
