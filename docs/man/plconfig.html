Name

plconfig - the Pipeline configuration and license generator program

Synopsis

plconfig [options]

plconfig --help
plconfig --html-help
plconfig --version
plconfig --release-date
plconfig --copyright

Description

This program is used by system adminstrators to generate a Pipeline configuration profile and license key for their site.  See the Global Options section for details about the site specific parameters which can be configured.

Before You Start

Before running this program, a new user called "pipeline" who's user group is also called "pipeline" needs to be created at your site.  The plcofig(1) program must be run as this new user.  The "pipeline" user will eventually be used to run various daemon processes and other priviledged programs which make up the Pipeline system.  This user should be able to login to all hosts which will run Pipeline programs with an idential user and group ID.  Once a site configuration profile has been generated using plcofig(1), this user and group ID should not be modified.

In addition, the root installation directory (see --root) for Pipeline needs to be created and have its file access permissions and ownership set before running plcofig(1) exactly as follows:

> mkdir root-install-directory
> chown root.pipeline root-install-directory
> chmod 775 root-install-directory

Where the (root-install-directory) is replaced with the location chosen for the Pipeline installation for your site.  Once these steps have been taken, you are ready to configure PIpeline for your site.

Configuring Pipeline

Once the steps described above, its a relatively simple matter to generate the Pipeline configuration and license key.  There are a small number of required options to plconfig(1) which are marked as such in the Global Options section.  The non-required options are either auto-detected by plconfig(1) or given reasonable default values.  The choices for all configuration parameters will be printed when plconfig(1) is run.  You may run plconfig(1) any number of times until your are satisfied with the settings. 

Each time plconfig(1) is run writes (or overwrites) the following three files under the root installation directory set with the --root-dir option:

root-install-directory/pipeline-license.key

This file contains the your license key for Pipeline.  This file is created with file access permissions which make it readable only by the "pipeline" user. This file should be zealously protected as it represents confidential information covered by the Pipeline License Agreement. Once Pipeline is installed, this file will be required in order to run the programs which make up the Pipeline.

root-install-directory/pipeline.profile

This file contains some encrypted text related the configuration options you gave to plconfig(1).  The contents of this file will need to be pasted into the profile section of the online Pipeline License Application Form.

root-install-directory/pipeline.config

This is a plain text file similar to the configuration parameters printed by plconfig(1).  This file is not required either to run Pipeline or as part of the license application process.  Its provided soley for your future reference of the configuration settings you chose when generating the other two files.

Once you've decided on a configuration, you should run plconfig(1) one last time adding the --lock option the rest of your option choices. This option make the above three file read-only to prevent any accidental destruction at a later date by running plconfig(1) again.

Information Options

The following options provide information about plconfig(1) and then immediately exit:
--help
Prints a brief usage message and exits.
--html-help
Opens this page using the mozilla(1) web browser. 
--version
Prints the version number of this software and exits.
--release-date
Prints date that this software was released and exits.
--copyright

Prints copyright notice for this software and exits.

Global Options

This options are used to set the configuration parameters of Pipeline for your site.  All of the Required Options must be supplied each time plconfig(1) is run.  For any of the Non-Required Options which are omitted, the values of the configuration parameters controlled by the option will either be auto-detected or default values will be used.

Required Options

--root-dir=dir

The absolute path to the root installation directory (dir) of Pipeline.  All programs and data files which make up Pipeline will live under this directory when Pipeline is installed.  This directory should reside on a network accessable file system such that all machines which will run Pipeline related programs can access the directory as (dir).

--node-dir=dir

--prod-dir=dir
The absolute path to the root directory under which all production data files will reside.  In other words, the location where artists will save their work.  This root production directory should reside on a network accessable file system such that all machines which will run Pipeline related programs can access the directory as (dir). PIpeline will create a standard set of subdirectories under this directory with the following structure:

dir/
  working/
    user-name/
      ...
    ...
  repository/
    ...
  checksum/
    ...

Where (dir) is the root production directory and (user-name) is the login name of an artist.  The "working" subdirectory contains working copies of files an artist is actively using.  The "repository" subdirectory contains read-only copies of the versions of files which have been checked-in using Pipeline's revision control features. The "checksums" directory contains checksum of the files in the other two subdirectories used to optimize some of Pipeline's file comparison features. 

It is possible to later move these subdirectories to other file systems using techiques such as symbolic links and mount points, but for optimal performance all of these subdirectories should reside on the same host which runs the file management daemon plfilemgr(1) (see --file-host). 

--home-dir=dir


--toolset-dir=dir


--master-host=host

--file-host=host
The hostname of the machine (host) which will run the plfilemgr(1) daemon.  This daemon should be run on the same host which acts as the network file server where production data files will reside on your site for optimal performance (see --prod-dir).

Non-Required Options

--master-port=num

--file-port=num
