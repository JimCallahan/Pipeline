Name

plput - copies working files into the repository

Synopsis

plput [--verbose] < filelist
plput --help
plput --html-help
plput --version
plput --release

Description

A utility program used internally by the pipeline(1) tool to copy files from a user's working area into the repository.  The files to be copied are read from stdin.  The input should consist of pairs of absolute filesystem paths seperated by whitespace.  The pairs specify the location in the working area and the repository location respectively.  

This utility will either succeed in copying all of the given files or fail to copy any of them.  Failure can occur if any of the source (working area) paths do not point to readable files.  Failure can also occur if any of the destination (repository) paths point to previously existing files.  In other words, this program will not ever destroy or overwrite repository files.

On success, the newly created repository files will owned by the "pipeline" user and have read-only (444) permissions regardless of which user invoked the program.  Also, any missing directories will be created in the repository as part of the copy and will also be owned by the "pipeline" user and have read-only access for all other users (755).

Options

--verbose
Print verbose progress and warning messages.
--help
Prints a brief usage message and exits.
--html-help
Opens this page using the mozilla(1) web browser.  Look here for details about how this works.
--version
Prints the version number of this software and exits.
--release
Prints date that this software was released and exits.
